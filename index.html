Our Crit Manifesto

  • Critiques are great!
  • Avoid "I" and "you" and talk about the work objectively.
  • No innocuous language: No, I like this, I hate this. Talk in constructive language. This color invokes calm, or this color is energetic.

In Class Today

Working in Groups
I'm assigning you a group of about 4 peers to work with. I'd like you to stick with this group for the entire term. They are your team. You'll help each other troubleshoot coding problems and critique each other's work.

The nature website on paper

  • The content of everything should take up about 80% of the page and be centered.
  • The header should have a background image that is full sized of a nice nature scene.
  • The level 1 heading should be whatever you want to call it. Or if you haven't had coffee yet, called it "Neature Nuggets" (and then look for neature nuggets on youtube fter class beecause it's hilarious).
  • The navigation should be a horizontal bar and have: Home, About, Blog, Contact. Each one of these shoud link to foo.html
  • The main content section should have a light cream colored background
  • All the fonts on the website should have a dark grey color and the links should be a dark red with a dark green hover.
  • The background color for the entire webpage should be a creamy green color.
  • The content area should have three columns. THe center column should be wider than the left and right columns.
  • The center column in the content area should have a level 2 header on top, about an amimal of your choice, followed by an animal image of that animal (from unsplash), and a bunch of copy. Below the copy, there should be a button that says "Read more" (it doesn't need a link)
  • The left-hand side should have an unordered linst of other animals with links to animal.html (all links can be the same for the sake of speed.)
  • The right hand side should just be of a level 3 header that says "resources" and a bunch of text.
  • finally there should be a footer that is in a dark color that has the font in white
  • Work together to hand code a website on a piece of paper with pencil. One person should be the designated googler. See above.
  • Project 1 meeting. Please meet and discuss each other's projects thus far. Ask questions. Stuff Like: How will the user get back to the homepage? Will there be some sort of nav on every page so you can jump around? Maybe arrows? Is a navigation needed at all? Have you thought about a single page design? How will you find solutions for X challenging thing to figure out how to code.
